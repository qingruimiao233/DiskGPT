# DiskGPT

**DiskGPT** 是一个专为 Linux + Windows 双系统用户设计的磁盘虚拟化拼接工具。

### 核心痛点
当你的 Linux（宿主机）和 Windows（本地磁盘中）处于**同一块物理硬盘**时， KVM 磁盘直通（Passthrough）无法将单个磁盘分区直通给虚拟机

### 解决方案
`diskgpt` 利用 Linux 内核的 `device-mapper` (dmsetup) 技术，动态创建一张“拼接映射表”：
1. **物理映射**：将你指定的 Windows 分区直接映射到虚拟盘的对应位置，确保原始数据读写。
2. **虚拟映射**：将 Linux 占用的分区或 GPT 表间隙映射到一个空的镜像文件（Sparse File）。
3. **最终产物**：在 `/dev/mapper/` 下生成一个完整的虚拟块设备，虚拟机可以像访问原生硬盘一样访问它，且不会破坏宿主机的分区。

---

## ✨ 主要功能

- **交互式分区选择**：自动扫描硬盘，用户只需输入序号（如 `3 4`序号之间需要使用空格分开）即可完成拼接。
- **配置持久化**：自动记忆上一次挂载的分区选择，下次运行一键直通。
- **GPU 唤醒支持**：集成显卡 PCI 重置逻辑（`remove` & `rescan`），解决直通显卡后常见的代码 43 或虚拟机死机问题。（这个说不准我本人的电脑上没问题）
- **VM 联动启动**：挂载完成后自动扫描并列出 KVM 虚拟机，支持记忆上次启动的虚拟机名称。
- **纯净二进制**：基于 C++ 编写，支持一键安装/卸载。

## 🛠️ 安装说明

### 环境依赖
在使用前，请确保系统已安装以下工具：
- `g++` (C++17 支持)
- `libvirt` / `virt-manager`
- `gdisk` (提供 `sgdisk` 用于同步分区表)
- `device-mapper` (提供 `dmsetup`)

### 编译与安装
在项目根目录下执行：
```bash
# 编译并安装到 /usr/local/bin
sudo make install
```

---

## 🚀 使用指南

### 启动挂载与虚拟机
```bash
sudo diskgpt -start
```
1. **选择分区**：首次运行时输入要直通的分区号（通常是 Windows 的 C 盘和 D 盘）。
2. **确认唤醒**：根据提示选择是否执行显卡唤醒指令。
3. **启动 VM**：选择要启动的虚拟机。

### 停止并清理环境
在关闭虚拟机后，建议运行此命令释放回环设备和映射表：
```bash
sudo diskgpt -stop
```

---

## ⚠️ 注意事项

1. **硬盘 ID**：程序当前硬编码为你的三星硬盘 ID。如果更换硬盘，请在 `main.cpp` 中修改 `DISK_ID` 变量。
2. **显卡 PCI 地址**：默认显卡唤醒地址为 `0000:01:00.0`。如果你的直通显卡地址不同，请在代码中相应位置进行微调。
3. **数据安全**：虽然本工具不会主动写入你的 Linux 分区，但操作磁盘映射具有一定风险，请确保在虚拟机配置中仅将生成的 `/dev/mapper/win11-spliced` 作为磁盘使用。

## 📂 文件说明

- `main.cpp`: 核心逻辑代码（C++）。
- `Makefile`: 自动化编译与安装脚本。
- `/etc/diskgpt_last_mount.conf`: 分区记忆文件（自动生成）。
- `/etc/diskgpt_last_vm.conf`: 虚拟机名记忆文件（自动生成）。
