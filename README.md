# DiskGPT

**DiskGPT** 是一个在KVM虚拟机中直通单个分区的磁盘虚拟化拼接工具。

### 核心痛点
当你的 Linux（宿主机）和 Windows（本地磁盘中）处于**同一块物理硬盘**时， KVM 磁盘直通（Passthrough）无法将单个磁盘分区直通给虚拟机

### 解决方案
`diskgpt` 利用 Linux 内核的 `device-mapper` (dmsetup) 技术，动态创建一张“拼接映射表”：
1. **物理映射**：将你指定的 Windows 分区直接映射到虚拟盘的对应位置，确保原始数据读写。
2. **虚拟映射**：将 Linux 占用的分区或 GPT 表间隙映射到一个空的镜像文件（Sparse File）。
3. **最终产物**：在 `/dev/mapper/` 下生成一个完整的虚拟块设备，虚拟机可以像访问原生硬盘一样访问它，且不会破坏宿主机的分区。

---

## ✨ 主要功能

- **交互式分区选择**：自动扫描硬盘，只需输入序号（如 `3 4`序号之间需要使用空格分开）即可完成拼接。
- **配置持久化**：自动记忆上一次挂载的分区选择，下次一键运行挂载。
- **GPU 唤醒支持**：目前来说可以自选显卡（这个说不准我本人的电脑上没问题）
- **VM 启动**：挂载完成后自动扫描并列出 KVM 虚拟机，能读取上次启动的虚拟机

---

## 🛠️ 环境依赖

程序内置了依赖项自检模块，运行前请确保系统中存在以下基础组件：
- 编译环境：`g++` (支持 C++17) 及 `make`
- 磁盘工具：`sgdisk` (来自 gdisk), `losetup`, `dmsetup`
- 调度工具：`lspci` (来自 pciutils), `virsh` (来自 libvirt)

### 编译与安装
在项目根目录下执行：
```bash
# 编译并安装到 /usr/local/bin
sudo make install
```

---

### 启动挂载向导
```bash
sudo diskgpt -start
```
程序将引导你完成以下流程：
1. **选择磁盘**：参考打印出的拓扑树，选择包含 Windows 系统的底层物理硬盘。
2. **选择分区**：输入需要映射的分区号（例如包含 C 盘和 D 盘的分区 `3 4`）。
3. **显卡唤醒 (可选)**：选择对应的独立显卡下发重置指令。
4. **启动 VM (可选)**：选择你的 Windows 虚拟机并拉起。


### 停止并清理环境
在关闭虚拟机后，建议运行此命令释放回环设备和映射表
其实你不清理也没啥事Linux不舒服会自己清理掉
```bash
sudo diskgpt -stop
```

### 自动挂载上一次配置
```bash
sudo diskgpt -automatic
```

---

## ⚠️ 注意事项
虽然这个玩意不会主动写入你的 Linux 分区，但操作磁盘映射具有一定风险，请确保在虚拟机配置中仅将生成的 `/dev/mapper/win11-spliced` 作为磁盘使用。

---

## 📂 配置文件位置

程序的记忆功能会生成以下纯文本配置文件，如需重置所有选择，直接删除它们即可：
- `/etc/diskgpt_last_disk.conf` (物理硬盘持久化 ID)
- `/etc/diskgpt_last_mount.conf` (挂载分区序列)
- `/etc/diskgpt_last_gpu.conf` (独立显卡 PCI 地址)
- `/etc/diskgpt_last_vm.conf` (KVM 虚拟机名称)
